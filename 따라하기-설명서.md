# SNPE 서면점 예약 사이트 — 왕초보 따라하기 설명서

이 설명서만 순서대로 따라 하면 사이트가 인터넷에 올라갑니다.
컴퓨터 용어가 어려워도 괜찮아요. **그대로 복사·붙여넣기** 하면 됩니다 🙂

---

## 📦 우리가 만든 것 (구조 한눈에 보기)

- **손님용 페이지** ( `내주소.com` ) → 로그인 없이 상담 예약 신청
- **관리자 페이지** ( `내주소.com/admin` ) → 비밀번호 넣고 신청 목록/시간표 관리
- **데이터베이스** (Supabase) → 손님 신청이 자동으로 저장되는 창고
- **배포** (Vercel) → 위 사이트를 인터넷에 무료로 올려주는 서비스

전체 흐름: **① Node 설치 → ② 이미지 넣기 → ③ Supabase(창고) 만들기 → ④ 비밀 값 넣기 → ⑤ 내 컴퓨터에서 미리보기 → ⑥ 인터넷에 배포**

> ⏱️ 처음이면 약 40~60분 걸립니다. 천천히 하셔도 돼요.

## 1단계. Node.js 설치 (프로그램 실행 도구)

1. 인터넷 주소창에 **https://nodejs.org** 접속
2. 초록색 버튼 두 개 중 **왼쪽 “LTS” (권장)** 버튼 클릭 → 설치 파일 다운로드
3. 다운로드한 파일 실행 → **계속 “다음(Next)”** 누르고 설치 (기본값 그대로)
4. 설치가 끝나면, **컴퓨터를 껐다 켜거나** 열려있는 창을 모두 닫았다 다시 여세요.
5. 확인: 시작 메뉴에서 **“PowerShell”** 검색 → 실행 → 아래를 입력하고 Enter
   ```
   node -v
   ```
   `v20.xxxx` 처럼 숫자가 나오면 성공입니다. ✅

## 3단계. Supabase(무료 데이터 창고) 만들기

### 3-1. 가입 & 프로젝트 만들기
1. **https://supabase.com** 접속 → 오른쪽 위 **“Start your project”** → GitHub 또는 이메일로 가입
2. **“New project”** 클릭
3. 입력:
   - **Name**: `snpe-seomyeon` (아무거나 OK)
   - **Database Password**: 비밀번호 생성 (아무거나, **어딘가 메모**해두세요)
   - **Region**: `Northeast Asia (Seoul)` 선택 (한국이라 빠름)
4. **“Create new project”** 클릭 → 1~2분 기다리면 완성

### 3-2. 표(테이블) 만들기 — SQL 복사 붙여넣기
1. 왼쪽 메뉴에서 **“SQL Editor”** 클릭
2. 우리 프로젝트 폴더의 **`supabase-setup.sql`** 파일을 메모장으로 열어 **전체 복사**
3. SQL Editor 빈칸에 **붙여넣기** → 오른쪽 아래 **“Run”** 클릭
4. 아래에 “Success” 비슷한 메시지가 나오면 성공 ✅ (표 4개가 만들어졌어요)

### 3-3. 비밀 값 2개 복사해두기
1. 왼쪽 아래 **톱니바퀴(Project Settings)** 클릭
2. **“Data API”** 메뉴 → **Project URL** 복사 → 메모장에 붙여두기
   - (`https://xxxxxxxx.supabase.co` 형태)
3. **“API Keys”** 메뉴 → **`service_role`** (secret) 항목의 **“Reveal” 후 값 복사** → 메모장에 붙여두기
   - ⚠️ 이 키는 **비밀번호 같은 것**! 아무한테도 보여주면 안 돼요.

> 지금 메모장에: ① Project URL, ② service_role 키 — 이렇게 두 개가 준비되면 됩니다.

## 4단계. 비밀 값 파일(.env.local) 만들기

1. 프로젝트 폴더에 있는 **`.env.local.example`** 파일을 복사해서
   같은 폴더에 **`.env.local`** 이라는 이름으로 붙여넣기 하세요.
   (윈도우에서 이름 바꿀 때 확장자가 `.txt`가 붙지 않게 주의!)
2. `.env.local` 을 메모장으로 열고, 아래처럼 **값을 채웁니다**:
   ```
   SUPABASE_URL=https://xxxxxxxx.supabase.co
   SUPABASE_SERVICE_ROLE_KEY=여기에-복사한-service_role-키
   ADMIN_PASSWORD=내가정한관리자비밀번호
   ```
   - `ADMIN_PASSWORD` 는 **관리자 페이지 로그인 비밀번호**예요. 원하는 걸로 정하세요. (예: `snpe2026!`)
3. 저장(Ctrl+S) 하고 닫습니다.

> 이 파일은 비밀 값이라 인터넷(GitHub)에 올라가지 않게 이미 막아뒀어요. 안심하세요.

## 5단계. 내 컴퓨터에서 미리보기 (로컬 실행)

1. **PowerShell** 실행
2. 아래를 한 줄씩 입력 (프로젝트 폴더로 이동):
   ```
   cd "C:\Users\LGPC\클로드 코드"
   ```
3. 필요한 부품 설치 (처음 한 번만, 1~3분 걸림):
   ```
   npm install
   ```
4. 사이트 켜기:
   ```
   npm run dev
   ```
5. `Local: http://localhost:3000` 같은 글씨가 나오면, 인터넷 브라우저 주소창에
   **http://localhost:3000** 입력 → 손님용 페이지가 보입니다! 🎉
6. 관리자 페이지는 **http://localhost:3000/admin** → 4단계에서 정한 비밀번호로 로그인
7. 끄고 싶으면 PowerShell에서 **Ctrl + C**

> 💡 화면이 안 뜨거나 에러가 나면 맨 아래 **“자주 겪는 문제”** 를 보세요.

## 6단계. 관리자 페이지 사용법

주소: `/admin` · 비밀번호: `.env.local` 에 넣은 `ADMIN_PASSWORD`

- **상담 신청**: 손님이 신청한 이름/연락처/불편한 곳/희망일/신청시각 목록. 삭제 가능.
- **수업 시간**: 홍혜경 원장님 / 묘니쌤 시간대 추가·수정·삭제.
- **상담 날짜**: 날짜+시간 추가, **‘마감/열기’ 토글**, 삭제. (요일은 자동 계산)
- **입금 계좌**: 은행/계좌번호/예금주 입력 후 저장 → 손님 신청 완료 화면에 표시됨.

> 처음엔 예시 데이터가 들어있어요. 관리자 페이지에서 지우거나 바꾸면 됩니다.

## 7단계. 인터넷에 배포하기 (Vercel · 무료)

배포는 **GitHub에 코드 올리기 → Vercel에 연결** 순서예요.

### 7-1. GitHub에 코드 올리기
1. **https://github.com** 가입/로그인 → 오른쪽 위 **+** → **“New repository”**
2. Repository name: `snpe-seomyeon` → **“Create repository”** (Public/Private 아무거나)
3. PowerShell에서 프로젝트 폴더로 이동 후 아래를 **한 줄씩** 실행:
   ```
   cd "C:\Users\LGPC\클로드 코드"
   git init
   git add .
   git commit -m "first commit"
   git branch -M main
   git remote add origin https://github.com/내아이디/snpe-seomyeon.git
   git push -u origin main
   ```
   - `내아이디` 부분은 본인 GitHub 아이디로 바꾸세요. (2번에서 만든 저장소 주소를 복사해 붙여도 됩니다.)
   - 중간에 GitHub 로그인 창이 뜨면 로그인하세요.

### 7-2. Vercel에 올리기
1. **https://vercel.com** → **“Sign Up”** → **GitHub 계정으로 가입** (제일 편함)
2. **“Add New… → Project”** → 방금 만든 `snpe-seomyeon` 저장소 **“Import”**
3. **⚠️ 중요! 환경 변수 넣기** — “Environment Variables” 칸에 아래 3개를 **하나씩** 추가:
   | Name | Value |
   |------|-------|
   | `SUPABASE_URL` | (3단계에서 복사한 Project URL) |
   | `SUPABASE_SERVICE_ROLE_KEY` | (3단계에서 복사한 service_role 키) |
   | `ADMIN_PASSWORD` | (내가 정한 관리자 비밀번호) |
4. **“Deploy”** 클릭 → 1~2분 기다리면 완료 🎉
5. 나오는 주소(예: `snpe-seomyeon.vercel.app`)가 **손님에게 보낼 사이트 주소**입니다.
   - 관리자: 그 주소 뒤에 `/admin` 붙이면 돼요.

### 7-3. 앞으로 내용 수정하려면?
- 관리자 페이지에서 바꾸는 건(시간표/날짜/계좌/신청목록) **바로 반영**돼요. 배포 다시 안 해도 됩니다.
- 디자인·글자 등 **코드**를 바꿨다면, PowerShell에서:
  ```
  git add .
  git commit -m "수정"
  git push
  ```
  → Vercel이 자동으로 다시 배포합니다.

## ❓ 자주 겪는 문제 (FAQ)

**Q. `node -v` 를 쳤는데 “인식할 수 없습니다” 라고 나와요.**
→ Node 설치 후 PowerShell을 껐다 다시 켜세요. 그래도 안 되면 컴퓨터 재시작.

**Q. `npm run dev` 후 화면에 에러가 나요.**
→ 대부분 `.env.local` 값이 잘못된 경우예요. URL/키에 **공백이나 따옴표**가 붙지 않았는지 확인하세요.

**Q. 관리자 로그인이 “비밀번호가 틀렸어요” 라고 해요.**
→ `.env.local`(로컬) 또는 Vercel 환경변수(배포)의 `ADMIN_PASSWORD` 와 정확히 같은지 확인. 값을 바꿨으면 `npm run dev` 를 껐다 켜야 반영됩니다.

**Q. 수강료 이미지가 안 보여요.**
→ 이미지 이름이 정확히 `price.png` 인지, `public` 폴더 안에 있는지 확인하세요.

**Q. 손님 신청이 관리자 목록에 안 보여요.**
→ Supabase의 SQL(3-2)이 정상 실행됐는지, `.env.local` / Vercel 환경변수의 URL·키가 맞는지 확인하세요.

**Q. 배포한 사이트에서만 데이터가 안 보여요.**
→ Vercel “Settings → Environment Variables” 에 3개 값이 다 들어갔는지 확인하고, 다시 **“Redeploy”** 하세요.
